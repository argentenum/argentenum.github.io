Digital Humanities Data Basics

Networks, metadata, linked data, Wikidata, and the Semantic Web

Network graphs

A network graph represents relationships between things.

Museum Artist Painting
↕
holds created by depicts
  • Things are shown as nodes.
  • Relationships are shown as edges.
  • Useful for seeing patterns, clusters, and connections.

Nodes and edges

Node

A node can be a person, place, object, book, or concept.

Example: museum, author, city

Edge

An edge is the relationship between two nodes.

Example: creator of, located in, part of

Node A connected to Node B

Sizing nodes and edges

Small node Medium node Large node
↕
Thin edge Normal edge Thick edge
  • Large nodes can show importance or number of connections.
  • Thicker edges can show stronger or more frequent relationships.
  • Visual size helps the viewer notice what matters most.

Colouring nodes and edges

Place Person Object
↕

  • Colour can group nodes by type, category, or cluster.
  • Edges can also be coloured by relationship type.
  • Good colour choices improve readability.

Classical metadata

Metadata is data about data.

Typical fields

  • Title
  • Creator
  • Date
  • Publisher

Why it matters

  • Find resources
  • Describe resources
  • Manage resources
  • Cite resources

Dublin Core

Dublin Core is a simple and widely used metadata standard.

Title Creator Subject Date Identifier
  • It gives a common vocabulary for describing resources.
  • It is simple, flexible, and widely adopted.

Linked data

Linked data publishes structured data on the web using identifiers and links.

URI → RDF → Linked data
  • Use URIs for things.
  • Use RDF to describe them.
  • Link to other datasets.

Triples

Linked data is built from triples:

Subject → Predicate → Object

Example: Vermeer → creator of → The Milkmaid

\[ \text{subject} \rightarrow \text{predicate} \rightarrow \text{object} \]

Linked open data

Linked open data is linked data that is also open for reuse.

Open

Shared licences and access.

Linked

Connected across datasets.

Reusable

Useful for research and integration.

Wikidata

Wikidata is a collaborative knowledge base of structured data.

Item + Property + Statement + Reference
  • Items identify things.
  • Properties describe relationships and values.
  • Statements connect items with data.

Ontology

An ontology describes concepts and relationships in a domain.

Museum is a type of Institution
  • Classes
  • Properties
  • Constraints
  • Meaning

Knowledge graph

A knowledge graph is a network of entities and relationships.

Entity ↔ Entity ↔ Entity
  • Data
  • Meaning
  • Links
  • Structure

Semantic Web

The Semantic Web aims to make web data understandable by machines.

Web documents → Linked data → Machine understanding

\[ \text{meaning on the web} = \text{data} + \text{links} + \text{shared vocabularies} \]

  • RDF
  • OWL
  • SPARQL
  • Linked data

Takeaway

  • Networks visualize relationships.
  • Metadata describes resources.
  • Linked data connects facts on the web.
  • Wikidata is a large linked knowledge graph.
  • Ontologies and the Semantic Web provide shared meaning.
